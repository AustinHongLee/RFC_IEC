# 焊口管理系統 (Weld Joint Management System)

把原本散在多個 Excel(焊口管制表、ISO 圖號清單)的資料,整合成一套以
**SQLite 為單一真實來源**的管理系統。後端 Python(FastAPI),前端 CSS+JS,
所有操作透過網頁 UI 登記,也支援 Excel 匯入 / 匯出。

層級:**專案 → 系統 → 管線 → 圖面(ISO) → 焊口**;焊口再連到檢驗(NDE)、
請款期別、問題、焊工、WPS 等。

---

## 一、怎麼啟動

需要先安裝 **Python 3.10 以上**(到 https://www.python.org 下載,安裝時務必勾選
「Add Python to PATH」)。

**Windows:** 直接雙擊 `run.bat`
**Mac / Linux:** 終端機執行 `./run.sh`

第一次啟動會自動建立虛擬環境並安裝套件(約 1 分鐘),完成後瀏覽器會自動打開
`http://127.0.0.1:8000`。要關閉系統,在黑色視窗按 `Ctrl + C` 即可。

> 已經幫你預載 5 個專案的資料(見下),打開就能直接看、直接操作。

## 三、各分頁功能

- **儀表板** — 焊口總數、完成率(口數 / DB數)、RT 合格率、圖面掃描、待處理問題,以及各系統 / 各銲接型式 / 狀態 / 請款期別的分佈。
- **圖面登錄** — 業主圖號 → 公司流水號(留空會自動配號)→ 掛焊口;含版次、預製掃描日期、狀態。
- **焊口登錄** — 核心。可搜尋、依狀態 / 系統篩選;點任一列開啟完整表單(規格、製程追溯、NDE、PWHT、試壓、商務),可加檢驗紀錄、用「推進▶」一鍵推進狀態。
- **請款期別** — 「依完成日自動歸期」會把每個焊口依完成日期歸入對應月份期別,統計各期 DB 數。
- **問題追蹤** — 欠接頭、新增焊口、圖面問題等,可結案 / 重開。
- **基礎資料** — 系統、焊工、WPS 維護。
- **匯入 / 匯出** — 匯入新的焊口管制表 Excel;或把目前專案焊口明細匯出成 Excel。
- **稽核日誌** — 每筆新增 / 修改 / 刪除 / 匯入都會留下紀錄(含操作人,請先在右上角填名字)。

## 四、匯入新專案

到「匯入 / 匯出」分頁,填專案代號、名稱,選 Excel 檔(.xlsx / .xlsm),按「開始匯入」。
系統會自動讀取「焊口編號明細」與「DWG NO.ALL / DRAWING LIST」分頁,辨識常見欄位
(圖號、銲口編號、尺寸、厚度、材質、銲接型式、係數、DB數、S/F、完成日期、CLASS、分類、RT…),
並自動把民國日期(如 114.06.05)轉成西元。

也可用指令匯入:
```
python importer.py "檔案路徑.xlsx" 專案代號 "專案名稱" "業主"
```

## 五、資料與備份

- 所有資料都存在這個資料夾的 **`weld.db`**(單一檔案)。要備份,複製這個檔即可。
- 重新匯入同一個專案代號,會**覆蓋**該專案的舊資料(其他專案不受影響)。
- 想重建示範資料:`python seed_template.py`。

## 六、檔案結構

```
焊口管理系統/
├─ app.py            後端 API + 啟動入口
├─ db.py             資料庫存取層(含稽核)
├─ schema.sql        資料庫結構定義
├─ importer.py       Excel 匯入器
├─ seed_template.py  建立 DEMO 範本資料
├─ weld.db           ★資料庫(你的所有資料都在這)
├─ static/           前端(index.html / app.js / style.css)
├─ run.bat / run.sh  一鍵啟動
└─ requirements.txt  套件清單
```

## 七、注意事項

- 設計給**單機或少數人**集中使用。若未來要十幾人同時連線即時編輯,可把後端
  改接 PostgreSQL 或做成正式網頁系統,**資料表結構可直接沿用**。
- 目前沒有帳號登入,操作人以右上角輸入的名字記錄在稽核日誌。
- SE-37 / gl-05 兩案的來源檔沒有乾淨的「系統」欄,所以系統分類會顯示「(未分類)」,
  其餘統計正常;可在「基礎資料」自行補系統。
- 開發 API 文件(自動產生)在 `http://127.0.0.1:8000/docs`。
