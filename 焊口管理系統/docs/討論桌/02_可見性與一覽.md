# 議題 02:可見性與一覽 — 2.5 萬個焊口,怎麼一眼看到重點

狀態:🟡 待各方意見　|　負責裁決:Claude

## 問題

打開系統,使用者**第一眼該看到什麼**?目前是一張密密麻麻的表格(只顯示前 300 筆),看不出「整體進度如何、哪裡卡住、今天該處理誰」。

核心問題:**怎麼讓人「看得見」——狀態、進度、待辦、異常,一眼就懂?**

## 使用者的話

> 直觀、可見性。

## 現況痛點

- 焊口清單就是一張表;狀態只有一個小徽章。
- 2.5 萬筆只顯示前 300,後面的「看不到」。
- 沒有「這張圖 / 這個 spool 完成到哪」的彙總視覺。
- 看不出「待完銲 / 待檢 / 待補焊 / 待試壓」各有多少、是誰的。

## 可能方向(拋磚)

1. **生命週期進度條**:每個焊口/圖面用一條 規劃→組對→完銲→檢驗→試壓→完成 的視覺條。
2. **卡片視圖**:重點焊口用卡片(放大關鍵欄位 + 顏色狀態),取代純表格。
3. **顏色/徽章規則**:狀態、S/F、NDE 合格/不合格、缺料、逾期,用一致的顏色語言。
4. **層級彙總**:圖面/spool 列直接顯示「總口數 / 已完成 / 待檢 / 完成率」。
5. **「我的待辦」面板**:待完銲、待檢、待補焊、待試壓,點進去就是篩好的清單。
6. **儀表板取捨**:現場最該天天看的 3–5 個數字是什麼?(完成率?待檢數?缺料?逾期?)

> 請各方:**對現場最有用的「可見性」是什麼**?別堆圖表,挑真正會被看的。
