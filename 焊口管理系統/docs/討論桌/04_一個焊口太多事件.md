# 議題 04:一個焊口太多事件 — 生命週期 vs 一張大表單

狀態:🟡 待各方意見　|　負責裁決:Claude

## 問題

一個焊口從頭到尾會經歷:設計 → 材料/組對 → 銲接 → 銲後/PWHT → NDE 檢測 → 壓力試驗 → 文件/請款/移交。
目前把這些**全部塞進一張 Modal 表單**,所以「讓人難受」。

核心問題:**怎麼呈現「一個焊口的一生」,讓人不被淹沒,又能在對的時機填對的東西?**

## 使用者的話

> 一個焊口需要解決那麼多事件……UI 根本讓人難受。

## 現況痛點

- 40 欄全攤在一張表單,不分階段、不分先後。
- 看不出「這個焊口走到哪一步、接下來該做什麼」。
- 設計/材料/銲接/檢驗/試壓/商務全混在一起。

## 可能方向(拋磚)

1. **分段工作流(stepper / 分頁)**:把表單拆成 概要 → 製程追溯 → 檢驗 → 試壓 → 商務,一次只看一段。
2. **事件時間軸(timeline)**:一個焊口主畫面用「發生過什麼」的時間軸呈現(組對→完銲→RT→補焊→合格→試壓),而不是一堆欄位框。狀態用「推進到下一步」按鈕帶動。
3. **核心 vs 延伸**:把欄位分「必填核心(編號/圖/尺寸/型式/狀態)」與「需要時才填」,平常只露核心。
4. **角色/工序導向**:銲工只碰銲接段、檢驗只碰 NDE 段,各自畫面乾淨。
5. **子表保留**:材料、檢驗已是「多筆子表」,維持子表(而非塞進主表)是否就是對的方向?還有哪些該變子表?

> 請各方:**「一張大表單」該變成什麼**?分頁 stepper、時間軸、還是核心+進階?講清楚一個現場人員打開一個焊口時,**最想先看到/先做的是什麼**。
