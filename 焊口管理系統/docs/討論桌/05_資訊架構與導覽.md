# 議題 05:資訊架構與導覽 — 分頁怎麼組織才直觀

狀態:🟡 待各方意見　|　負責裁決:Claude

## 問題

目前上方分頁是:**儀表板 / 圖面登錄 / 焊口登錄 / Spool 分段 / 請款期別 / 問題追蹤 / 基礎資料 / 匯入匯出 / 稽核**。
這是**照「資料表」排的**,不是照「使用者要做的事」排的。非工程師打開,不一定找得到自己要的。

核心問題:**導覽該怎麼組織,讓人 30 秒內找到並做到他要做的事?**

## 使用者的話

> 直觀。

## 現況痛點

- 分頁照資料表分(圖面/焊口/spool/請款…),不是照任務分。
- 主入口到底是「焊口」還是「圖面 / spool」?現場心智模型是哪個沒對齊。
- 9 個分頁一字排開,輕重不分。

## 可能方向(拋磚)

1. **以任務組織 vs 以資料表組織**:例如改成「今天要做 / 找焊口 / 看進度 / 設定」幾個大區。
2. **主入口選擇**:現場是「先想到圖號」還是「先想到焊口」還是「先想到某條 line / 某個 spool」?入口應對齊那個心智模型。
3. **精簡導覽**:把「基礎資料 / 匯入匯出 / 稽核」收進「設定/管理」次選單,主畫面只留高頻的。
4. **裝置情境**:會不會有人拿平板在現場用?要不要考慮觸控/大按鈕。
5. **首畫面**:第一次打開的 30 秒,最該能做到的一件事是什麼。
