# 議題 03:篩選與查找 — 想找什麼就能精確篩到

狀態:🟡 待各方意見　|　負責裁決:Claude　|　★ 使用者特別在意這題

## 問題

使用者明講:「假設有人想找什麼,可更精確的套入篩選。」目前篩選只有 **搜尋框 + 狀態 + 系統** 三個,遠遠不夠。

核心問題:**怎麼讓人用任意條件組合,精確找到那一批焊口?**

## 使用者的話

> 完整相關篩選、想找什麼可更精確的套入篩選。

## 現況痛點

- 只能篩:關鍵字(銲口/圖號/材質)、狀態、系統。
- 篩不了:尺寸、厚度、材質、銲接型式、S/F、焊工、NDE 方法/結果、補焊、完銲日期區間、spool、請款期別、承包商、分類…
- 不能「組合」(例:316L + 6" 以上 + 已完銲但未檢 + A 焊工)。
- 找到後不能直接批次處理 / 匯出。

## 可能方向(拋磚)

1. **多面向(faceted)篩選**:左欄一排可勾選的面向(系統、尺寸、材質、型式、S/F、狀態、焊工、NDE、日期區間、spool、期別…),可任意組合。
2. **每個面向顯示數量**:像購物網站(「316L (1,204)」),選之前就知道有多少。
3. **點選式 vs 搜尋語法**:非工程師多半偏「點選/勾選」;是否也保留一個強力搜尋框(`material:316L size:>6 status:完銲`)給進階者?
4. **儲存常用篩選**:把一組條件命名收藏成「視圖」(例:本月待檢、外包未請款)。
5. **量大處理**:2.5 萬筆的排序、分頁 / 虛擬捲動、即時計數效能。
6. **篩完即動作**:篩出來能直接批次填值 / 匯出 Excel。

> 請各方:**面向式篩選的最小可用集合**是哪幾個面向?排序與預設怎麼設?點選式與語法式如何取捨?
