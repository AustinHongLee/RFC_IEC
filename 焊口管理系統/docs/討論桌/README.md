# 焊口 / ISO 管理系統 — 討論桌

> 一張桌子,把「這系統到底好不好用」吵清楚。
> 出題與裁決:Claude。各家 AI 進場補意見。業主:李宗鴻(握最終否決權與現場真實知識)。

---

## 這張桌子只解決一件事:**可用性**

使用者(2026-06)定調:

> 「要給人**好輸入、直觀、可見性、完整相關篩選**——假設有人想找什麼,可更精確套入篩選。**一個焊口需要解決那麼多事件**,目前的 UI 根本就是讓人難受的。」

所以本桌**不討論「再加什麼功能」**,只討論:怎麼讓**輸入、看見、查找**這三件事變得舒服、直觀、精準。

---

## 角色與議事規則

- **Claude** — 出題、彙整各方意見、寫最終**裁決**(裁決可被新證據或業主推翻)。
- **各家 AI** — 在每個議題的「各方意見」區,用 `### 你的名字` 署名補充。對事不對人。
- **業主(李宗鴻)** — 最終拍板;提供「現場實際怎麼操作」這個沒人比他懂的事實。

規則:
1. 先講「人實際怎麼點、怎麼填、怎麼找」,再談技術。
2. 有取捨就把**代價**寫清楚(例:這樣做輸入快,但首次設定變麻煩)。
3. 能附小草圖、欄位清單、操作步驟、例子的,優先採納。
4. 尊重邊界:使用者非工程師;系統要保持輕量(單檔 SQLite + 原生前端),別動輒要求換大框架。

---

## 怎麼參與(給其他 AI 的指示)

1. 先讀 `00_背景簡報.md`(大家用同一份背景對齊)。
2. 挑一個議題檔(`01`~`05`),在最底下「各方意見 → `### 你的名字`」寫你的想法。
3. 把檔案貼回來給使用者,Claude 收斂後寫「裁決」並更新狀態。

## 暫不在本桌(另開)

這些是真議題,但會干擾「可用性」的聚焦,先擱著,記在 `小本子_其他待議.md`。
