# 議題 01:輸入體驗 — 怎麼讓「登記/更新一個焊口」快又不痛苦

狀態:🟡 待各方意見　|　負責裁決:Claude

## 問題

一個焊口有 ~40 個欄位。大量初始資料靠 Excel 匯入沒問題;但**日常更新是逐口或逐批發生的**——今天填一批「完銲日期 + 焊工」、明天填「NDE 結果」、後天補「材料爐號」。目前每次都要開一張塞滿 40 欄的長表單,**慢、亂、容易填錯欄**。

核心問題:**怎麼讓現場人員把「常做的那幾種登記」做到又快又不出錯?**

## 使用者的話(北極星)

> 好輸入、直觀、一個焊口那麼多事件不要一次全丟給我。

## 現況痛點

- 一個 Modal = 40 欄全攤開,不分輕重、不分時機。
- 沒有批次:100 個焊口同一天完銲、同一個焊工,要開 100 次表單。
- 沒有預設:同一條 line 的材質 / WPS / 焊工其實都一樣,卻每口重填。
- 不相關的欄位也擋在眼前(螺紋接頭根本不用 PWHT / NDE)。

## 可能方向(拋磚,不是定論)

1. **漸進式揭露**:平常只顯示最常用的 6–8 欄,其餘收進「進階 ▸」。
2. **依型式帶預設 + 只顯示相關欄位**:選了 BW 才出現 NDE/PWHT;螺紋就不顯示。
3. **批次操作**:清單勾選多個焊口 → 一次填「完銲日期 / 焊工 / 狀態 / NDE 結果」。
4. **行內編輯(inline)**:清單上直接點某格改值,不開 Modal。
5. **鍵盤流**:Tab 走位、Enter 存並跳下一筆、「複製上一筆」。
6. **工序模式**:選「我在做什麼」(銲接登記 / 檢驗登記 / 材料登記)→ 只給該工序要填的欄位。
7. **範本 / 套用**:把一條 line 或一張圖的共通值(材質、WPS、焊工)套到底下所有焊口。

> 請各方:不必全包,**挑你認為對「非工程師、現場、量大」最有效的 1–2 個**,講清楚實際操作步驟與代價。
