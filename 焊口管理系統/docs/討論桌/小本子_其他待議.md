# 小本子 — 其他待議(暫不在討論桌焦點)

這些是真實但本輪先擱置的議題,別忘了。等「可用性」這桌收斂後再開。

## 1. 資料來源誰為準(SSOT)
Excel / 資料庫 / UI 三邊。
**目前「重新匯入同一個專案代號」會覆蓋整個專案 → 會洗掉在 UI 手動輸入的資料(spool、材料、檢驗紀錄、手改的狀態…)。**
一旦開始認真用 UI 登記,這個衝突一定要先解(例:匯入只補新焊口、不覆蓋既有;或明確「DB 才是唯一真實來源,Excel 只進不出」)。

## 2. 資料安全與備份
weld.db(單一檔案)目前**有被 git 追蹤**,用 `push.ps1`(`git add -A` + push)當備份。
風險:app 開著時備份 → 可能備到「寫到一半」的毀損檔(已實際發生過一次)。
待決:要嘛「關 app 再備份」當紀律;要嘛 **DB 不進 git**,改用「匯出 Excel / 複製到 backup/」當備份。

## 3. 單機 SQLite 的壽命
何時需要升級成多人同時即時編輯(PostgreSQL / 正式網頁系統)?目前單人/少數人 OK。資料表結構已設計成可直接沿用。
