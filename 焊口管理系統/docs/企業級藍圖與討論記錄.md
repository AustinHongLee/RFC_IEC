# 焊口管理系統 — 企業級藍圖與討論記錄

> 記錄於 2026 年 6 月的設計討論。用途:隔天接續開發的依據與規格。

## 0. 一句話

把焊口 / 圖面從「散落的 Excel」進化成「以 SQLite 為單一真實來源、用 UI 登記、可跨案彙總」的企業級系統。本文記錄完整生命週期定義、DB數 的確切定義、以及升級路線圖。

---

## 1. 目前已完成(v1)

- SQLite + FastAPI 後端 + 原生 CSS/JS 前端,放在 `焊口管理系統/`,`run.bat` 一鍵啟動。
- 層級:**專案 → 系統 → 管線 → 圖面(ISO)→ 焊口**;旁接 圖面版次 / 焊工 / WPS / 檢驗(NDE)/ 請款期別 / 問題 / 稽核。
- 已匯入 5 案:DEMO(9)、cp-129(5,379)、SE-37(471)、gl-05(5,852)、se-39(24,879)。
- 功能:儀表板、圖面登錄、焊口登錄(完整企業欄位)、檢驗、請款自動歸期、問題追蹤、匯入 / 匯出、稽核日誌。

## 2. DB數 的定義(已用全資料驗證)

- **結論:DB數 就是吋徑(diameter-inch),公司慣用名,保留、不改名。**
- 公式(cp-129 全 5,379 口 100% 符合):

  **DB數 = max(1, 尺寸吋) × 係數**

  - 小於 1" 的(½"、¾")計為 1;≥1" 保留實際值(1.5" 就是 1.5)。
  - 係數 = 銲接型式加權,本案全部為 1。
- 系統現況:已有 `db_count`(DB數)與 `db_factor`(係數),進度與請款都以 DB數 加總。
- 待辦:UI / 匯入新增焊口時,DB數 留空則自動帶 `max(1, 尺寸) × 係數`。

## 3. 一個焊口的全生命週期(0 → 結束)

| 階段 | 該蒐集的資訊 |
|---|---|
| **0 定義 / 設計** | 焊口唯一編號(W-001)、所屬 ISO圖 / spool / 張次、line no、系統、區域;接頭型式(BW/SW/承插/分歧)、尺寸 NPS、SCH / 壁厚、兩側母材規格與 P-No、配管等級、流體類別(B31.3 Normal/Cat.M/高壓);指定 WPS、NDE 種類與比例、PWHT、硬度 / PMI、預製 S 或現場 F |
| **1 材料 / 組對** | 兩側元件爐號 heat no、MTR 材質證明、PMI;銲材 AWS class / F-No / 批號、保護氣 / 背檔氣;組對者、日期、間隙對齊、fit-up 目視合格 |
| **2 銲接** | 焊工鋼印(打底 / 填充 / 蓋面)、資格連續性;WPS、實際製程(GTAW/SMAW…)、預熱 / 層間溫度、熱輸入;銲接日期、班別、現場天候 |
| **3 銲後 / PWHT** | 目視 VT 結果;PWHT(若需)曲線圖號 / 溫度 / 時間 / 升降溫率 / 日期;硬度、PMI |
| **4 NDE 檢測** | 方法 RT/UT/PT/MT/PAUT、比例、技術、委外單位;報告號、合格 / 不合格、缺陷類型與位置、檢測員;不合格 → 補焊紀錄、補焊次數、重檢 |
| **5 壓力試驗** | 試壓包編號、種類(水壓 / 氣壓)、壓力、介質;日期、結果、復原 reinstatement、punch |
| **6 文件 / 請款 / 移交** | 納入 weld map / as-built、MDR 卷冊;吋徑 DB數、進度 %、請款期別、單價;最終狀態、歸檔 |

## 4. 一個圖面(ISO)的全生命週期(0 → 結束)

| 階段 | 該蒐集的資訊 |
|---|---|
| **0 接收 / 登錄** | 業主圖號、內部流水號、版次、收圖日、transmittal;line no、系統、區域、P&ID、pipe class、尺寸、第 x / 共 y 張、圖種 |
| **1 審圖 / 取量** | MTO 物料清單;**spool 分解(廠內 shop spool + 現場 field weld)**、spool 編號、weld map(shop/field 口數);版次歷程、作廢取代 |
| **2 採購 / 料控** | 請購 PR → 採購 PO → 收料 GRN → 倉儲 → 領料;爐號 / MTR 連結、自購 vs 業主供料;料況(已訂 / 到料 / 缺料)、缺料預警 |
| **3 預製(廠內)** | 各 spool:下料 / 組對 / 銲接 / NDE / PWHT / 油漆 狀態;預製圖掃描、fab dwg 發行、spool 完成 |
| **4 發行施工 / 現場** | IFC 發行、運送到場;現場組立 erection、field weld、安裝進度 |
| **5 檢驗 / 試壓** | 全圖焊口 NDE 完成、指派試壓包;水壓 / 氣壓、punch list |
| **6 竣工 / 結案** | 紅線 redline / as-built、最終版次;移交卷冊、圖面狀態 = 完成 / 歸檔 |

## 5. 企業延伸

材料追溯(MTR / 爐號 / PMI)、採購鏈(MTO→PR→PO→收料→領料)、銲材 / 消耗品(烘箱紀錄)、焊工資格(WPQ/PQR/WPS + 連續性)、NDE 廠商與報告、油漆塗裝 / 保溫、試壓包管理、品保卷冊(MDR / turnover)、成本與吋徑計量、外包計價。

---

## 6. 公司視角:管線預製 + 現場安裝

核心策略:**最大化廠內 shop weld**(品質高、成本低、易檢),**最小化現場 field weld**(慢、貴、難檢)。需要建立的能力:spool 管理(廠內編號)、weld map 分 shop / field、材料控制(MTO→領料 + 爐號追溯)、焊工連續性、WPS/PQR 庫、NDE 協調與歸檔、油漆塗裝、出貨物流(spool 運工地)、現場組立追蹤、試壓包 + punch、以吋徑 DB數 計進度與請款。

---

## 7. 升級路線圖(建議順序)

1. **spool 層** — 圖面 → spool → 焊口,焊口標 shop/field。這是預製 + 安裝公司的核心維度(目前系統還沒有)。
2. **DB數 自動計算** — `max(1, 尺寸) × 係數`,UI 與匯入皆套用。
3. **材料 / MTR 追溯** — 爐號 → 焊口兩側、銲材批號。
4. **採購模組** — MTO→PR→PO→收料→領料,料況看板 / 缺料預警。
5. **試壓包 + 品保卷冊(MDR / turnover)**。
6. **資料補強** — se-39 完成資料(在「完成焊口建檔」分頁)合併進來;ISO 圖號清單的「預製掃描日期」接入圖面。
7. **(未來)多人同時** — 升級 PostgreSQL 或正式網頁系統,資料表結構可直接沿用。

---

## 8. 已知資料來源限制

- **se-39**:焊口編號明細沒有完成日期(在別的分頁),所以完成率暫顯 0%,待第 6 點合併後修正。
- **SE-37 / gl-05**:來源檔沒有乾淨的「系統」欄,系統分組顯示「未分類」;可在「基礎資料」自行補,或之後由 line no 前綴推導。
